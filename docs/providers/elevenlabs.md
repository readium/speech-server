# Provider: ElevenLabs

Model-level information for the `elevenlabs` provider — a **hosted HTTP** TTS API (no local
models, no torch). The server proxies synthesis requests to ElevenLabs and returns the audio.

Source: [ElevenLabs Text-to-Speech](https://elevenlabs.io/docs/capabilities/text-to-speech).

## Identity

- **Provider id:** `elevenlabs`
- **Voice identifier:** `urn:readium:tts:elevenlabs:<slug>` (e.g. `urn:readium:tts:elevenlabs:george`).
  `originalName` is a readable handle (e.g. `george`). The ElevenLabs API `voice_id` is mapped
  **in code** — `_VOICE_IDS` in `app/providers/elevenlabs.py`, keyed by `originalName` — so
  `voices.json` stays a readable catalog with no opaque ids. Add a `_VOICE_IDS` entry per voice.

## Configuration

Provider-scoped config lives in **`elevenlabs.env`**, generated by the setup wizard
(`scripts/configure.sh` → **Manage provider → elevenlabs**). Universal keys
(`ENABLED_PROVIDERS`, auth, concurrency) stay in `.env`.

| Key | Default | Notes |
|---|---|---|
| `ELEVENLABS_API_KEY` | — | **Required** when `elevenlabs` is in `ENABLED_PROVIDERS`. From elevenlabs.io → profile → API key. |
| `ELEVENLABS_MODEL_ID` | `eleven_multilingual_v2` | See model comparison below. |
| `ELEVENLABS_LANGUAGES` | — | ElevenLabs' **own** language set (comma BCP-47 prefixes, e.g. `en,hi`), independent of pocket's `LANGUAGES`. Empty = no ElevenLabs voices served. Wizard: **Manage provider → elevenlabs → Set languages**. |
| `ELEVENLABS_BASE_URL` | `https://api.elevenlabs.io` | Override for testing. |

## Models & cost

**Billing is per character, and the model sets the multiplier** — output format never changes cost.
Pick a model for the cost/quality/latency trade-off:

| Model | Languages | Best for |
|---|---|---|
| `eleven_flash_v2_5` | 32 | Cheapest + lowest latency. |
| `eleven_turbo_v2_5` | 32 | Cheapest with better quality than flash — good default for read-aloud. |
| `eleven_multilingual_v2` | 29 | Solid all-round quality. |
| `eleven_v3` | 70+ | Highest quality / most expressive, most languages. |

Compare and confirm current pricing yourself (ElevenLabs changes it):

- **Models:** <https://elevenlabs.io/docs/models>
- **Pricing / credit costs:** <https://elevenlabs.io/pricing>

## Audio output

ElevenLabs encodes **server-side** — the requested format comes back already encoded, so the
server skips its own ffmpeg step (unlike PocketTTS).

- `mp3` — `mp3_44100_128` (44.1 kHz). **192 kbps needs the Creator tier**, so it's capped at 128.
- `opus` — `opus_48000_128` (48 kHz, ogg/opus).
- `wav` — `wav_24000` (24 kHz; `wav_44100` needs the Pro tier).

All variants above are free-tier-safe. Native `mp3`/`opus` run at 44.1/48 kHz — higher fidelity than
raw `pcm_24000` would give on a non-Pro plan. Word-level **boundary marks** are returned for every
format (via the `/with-timestamps` endpoint).

## Languages & voices

- **Common-to-all-models set** — ElevenLabs' models can do 29–70+ languages, but we deliberately do
  **not** mirror the full per-model catalog. The ceiling is the 29 languages **common to every
  model** (`multilingual_v2 ∩ flash/turbo_v2.5 ∩ v3`), so it's model-independent — the `_SUPPORTED_LANGUAGES`
  constant in `app/providers/elevenlabs.py`. All are free-tier usable (free tier paywalls Voice-Library
  *voices*, not *languages*). A voice serves this whole set unless it narrows via `otherLanguages`.
  When a model's extra languages (v2.5 `hu/no/vi`, v3's 40+ more) are actually wanted, reintroduce a
  per-model `models.json` keyed by `model_id`. No per-voice install cost (unlike PocketTTS embeddings).
- The served catalog is `app/data/voices/elevenlabs/voices.json` — a **lean, operator-curated**
  list, not a mirror of ElevenLabs' voice library. It ships with a small default set; add or remove
  voices to suit the deployment. Each entry has a `language` (accent) plus `otherLanguages` — the
  default voices list the supported set (`de es fr it pt`); leave it empty and the voice falls back
  to `_SUPPORTED_LANGUAGES`, or narrow it to **restrict** a voice (e.g. a language-locked cloned
  voice). `GET /voices` returns the resulting per-voice set intersected with `ELEVENLABS_LANGUAGES`.
- **Author the catalog from your account** — dump your account's voices as a starting point, then
  **trim** to the ones you want served:

  ```bash
  ELEVENLABS_API_KEY=sk_... python scripts/fetch_elevenlabs_voices.py           # print, review
  ELEVENLABS_API_KEY=sk_... python scripts/fetch_elevenlabs_voices.py --write   # overwrite, then curate
  ```

## Constraints

- No local models — needs network access to `api.elevenlabs.io` and a valid API key.
- Subject to ElevenLabs rate limits (a `429` surfaces as `rate_limited`) and your plan's character
  quota. The per-provider circuit breaker trips on repeated **5xx** failures (client/quota errors
  like `429`/`404` don't open it).
- **Daily budget (spam guard):** `ELEVENLABS_DAILY_CHAR_LIMIT` caps **characters** sent per day;
  `0` = unlimited. Kept simple on purpose — set it per your model's rate from
  [pricing](https://elevenlabs.io/pricing/api) (Flash/Turbo $0.05 vs Multilingual/v3 $0.10 per 1K
  chars, so flash/turbo bill half → allow ~2× the characters for the same cost). Over the cap,
  `/synthesize` returns `429 rate_limited` until 00:00 UTC. The counter is a small JSON file
  (`ELEVENLABS_USAGE_FILE`, default `/tmp/…`) shared across workers — point it at a volume to persist
  across restarts. Set it in the wizard (**Manage provider → elevenlabs → Set daily limit**).
- `controls`: `speed` (maps to `voice_settings.speed`) and `boundary` supported; `pitch` and `ssml`
  are not (SSML tags are stripped).
